# FastAPI Path Parameters and Query Parameters

A clear and practical guide to understanding **Path Parameters** and **Query Parameters** in FastAPI and REST APIs.

This document explains:

* What they are
* Where they are used
* Why they are used
* How they work in FastAPI
* Real API examples

---

# 1. Introduction

When building APIs, the client (browser, mobile app, or another service) needs to send information to the server.

The most common way to send information in a URL is through:

* **Path Parameters**
* **Query Parameters**

Both are part of the URL but serve **different purposes in API design**.

Understanding the difference is **very important for backend, ML, and MLOps engineers building APIs**.

---

# 2. Example URL Structure

Example API request:

```
GET /users/10?details=true
```

Breakdown:

| Part           | Type            | Meaning                      |
| -------------- | --------------- | ---------------------------- |
| `/users/10`    | Path Parameter  | Identifies the specific user |
| `details=true` | Query Parameter | Controls what data to return |

---

# 3. Path Parameters

## What are Path Parameters?

**Path parameters are dynamic values embedded inside the URL path.**

They are used to **identify a specific resource**.

Example URL:

```
/users/10
```

Here:

```
10 → path parameter
```

This means:

```
Give me the user whose ID is 10
```

---

# 4. FastAPI Example: Path Parameter

```python
from fastapi import FastAPI

app = FastAPI()

@app.get("/users/{user_id}")
def get_user(user_id: int):
    return {"user_id": user_id}
```

---

# 5. How Path Parameters Work

Client request:

```
/users/5
```

FastAPI extracts the value:

```
user_id = 5
```

Function executed:

```python
get_user(user_id=5)
```

Response:

```json
{
  "user_id": 5
}
```

---

# 6. Automatic Type Validation

FastAPI uses **Python type hints** to validate inputs.

Example:

```python
def get_user(user_id: int):
```

If the user sends:

```
/users/abc
```

FastAPI automatically returns an error because `abc` is not an integer.

Example error:

```json
{
  "detail": [
    {
      "type": "int_parsing",
      "msg": "Input should be a valid integer"
    }
  ]
}
```

---

# 7. Where Path Parameters Are Used

Path parameters are used when we want to **identify a specific resource**.

Examples:

| Resource               | Example URL    |
| ---------------------- | -------------- |
| Get specific user      | `/users/10`    |
| Get specific product   | `/products/25` |
| Get specific order     | `/orders/500`  |
| Get specific blog post | `/posts/101`   |

Meaning:

```
Give me THIS specific resource
```

---

# 8. Multiple Path Parameters

FastAPI supports multiple path parameters.

Example:

```python
@app.get("/users/{user_id}/orders/{order_id}")
def get_order(user_id: int, order_id: int):
    return {
        "user_id": user_id,
        "order_id": order_id
    }
```

Request:

```
/users/10/orders/500
```

Response:

```json
{
  "user_id": 10,
  "order_id": 500
}
```

Meaning:

```
Give me order 500 belonging to user 10
```

---

# 9. Query Parameters

## What are Query Parameters?

Query parameters are **values added after `?` in the URL**.

Example:

```
/products?limit=10
```

Here:

```
limit → query parameter
```

---

# 10. FastAPI Example: Query Parameter

```python
@app.get("/products")
def get_products(limit: int):
    return {"limit": limit}
```

Request:

```
/products?limit=10
```

Response:

```json
{
  "limit": 10
}
```

---

# 11. Where Query Parameters Are Used

Query parameters are used for:

* Filtering
* Searching
* Pagination
* Sorting
* Optional options

Examples:

| Purpose    | Example                          |
| ---------- | -------------------------------- |
| Pagination | `/products?limit=10&skip=20`     |
| Search     | `/products?name=laptop`          |
| Filtering  | `/products?category=electronics` |
| Sorting    | `/products?sort=price`           |

---

# 12. Why Query Parameters Are Used

Query parameters **modify how the data is returned**, not which resource it is.

Example:

```
/products
```

Meaning:

```
Return all products
```

Example:

```
/products?limit=10
```

Meaning:

```
Return first 10 products
```

---

# 13. Multiple Query Parameters

```python
@app.get("/products")
def get_products(limit: int, skip: int):
    return {
        "limit": limit,
        "skip": skip
    }
```

Request:

```
/products?limit=10&skip=5
```

Response:

```json
{
  "limit": 10,
  "skip": 5
}
```

---

# 14. Optional Query Parameters

Query parameters can have **default values**, making them optional.

```python
@app.get("/products")
def get_products(limit: int = 10):
    return {"limit": limit}
```

Request:

```
/products
```

Response:

```json
{
  "limit": 10
}
```

---

# 15. Query Parameter with None

```python
@app.get("/items")
def get_items(q: str = None):
    return {"query": q}
```

Request:

```
/items?q=laptop
```

Response:

```json
{"query": "laptop"}
```

Request without query parameter:

```
/items
```

Response:

```json
{"query": null}
```

---

# 16. Using Path and Query Parameters Together

Example FastAPI code:

```python
@app.get("/users/{user_id}")
def get_user(user_id: int, details: bool = False):
    return {
        "user_id": user_id,
        "details": details
    }
```

Example request:

```
/users/10?details=true
```

Response:

```json
{
  "user_id": 10,
  "details": true
}
```

Explanation:

| Parameter | Type            | Meaning         |
| --------- | --------------- | --------------- |
| `user_id` | Path Parameter  | Identify user   |
| `details` | Query Parameter | Optional option |

# 17. Real API Example

Example API request:

```
GET /products/10?discount=true
```

Meaning:

| Component       | Meaning                 |
| --------------- | ----------------------- |
| `/products/10`  | Product ID              |
| `discount=true` | Return discounted price |

FastAPI implementation:

```python
@app.get("/products/{product_id}")
def get_product(product_id: int, discount: bool = False):

    if discount:
        return {
            "product_id": product_id,
            "price": 90
        }

    return {
        "product_id": product_id,
        "price": 100
    }
```

Requests:

```
/products/5

/products/5?discount=true
```

---

# 18. Path vs Query Parameters (Very Important)

| Feature     | Path Parameter             | Query Parameter               |
| ----------- | -------------------------- | ----------------------------- |
| Location    | Inside URL path            | After `?`                     |
| Example     | `/users/10`                | `/users?limit=10`             |
| Purpose     | Identify specific resource | Modify returned result        |
| Required    | Usually required           | Usually optional              |
| Typical use | Fetch single item          | Filtering, search, pagination |

---

# 19. Easy Way to Remember

## Path Parameter

Used for **identity**.

Examples:

```
/users/10
/products/5
/orders/100
```

Meaning:

```
Give me THIS specific thing
```

---

## Query Parameter

Used for **options**.

Examples:

```
/products?limit=10
/products?category=electronics
/products?sort=price
```

Meaning:

```
Modify how the results are returned
```

---

# 20. Final Concept

**Path Parameter**

```
Identifies WHICH resource
```

**Query Parameter**

```
Controls HOW the data is returned
```
