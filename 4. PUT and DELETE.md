# FastAPI PUT and DELETE Methods

---

# 1. Introduction

When building APIs with **FastAPI**, different HTTP methods are used to perform different actions on resources.

In REST APIs, the most common operations are known as **CRUD operations**:

| Operation | HTTP Method | Description           |
| --------- | ----------- | --------------------- |
| Create    | POST        | Create a new resource |
| Read      | GET         | Retrieve data         |
| Update    | PUT / PATCH | Update existing data  |
| Delete    | DELETE      | Remove data           |

In this guide we focus on:

* **PUT method** (Update data)
* **DELETE method** (Remove data)

---

# 2. FastAPI PUT Method

## Concept

The **PUT method is used to update an existing resource** on the server.

Important characteristics:

* Used to **replace existing data**
* The entire resource is usually updated
* Data is sent through the **request body (JSON)**

Typical real-world use cases:

* Update user profile
* Update product details
* Update order information

---

# 3. Example — PUT API

```python
from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI()

items = {
    1: {"name": "Laptop", "price": 50000},
    2: {"name": "Phone", "price": 20000}
}

class Item(BaseModel):
    name: str
    price: float

@app.put("/items/{item_id}")
def update_item(item_id: int, item: Item):
    items[item_id] = item
    return {
        "message": "Item updated",
        "item_id": item_id,
        "data": item
    }
```

---

# 4. How the PUT Request Works

### Step 1 — Client Sends Request

```
PUT /items/1
```

### Request Body

```json
{
 "name": "Gaming Laptop",
 "price": 75000
}
```

---

### Step 2 — FastAPI Processes Request

FastAPI will:

1. Read the JSON body
2. Validate it using the **Pydantic model**
3. Replace the existing item data

---

### Step 3 — Server Response

```json
{
 "message": "Item updated",
 "item_id": 1,
 "data": {
  "name": "Gaming Laptop",
  "price": 75000
 }
}
```

---

# 5. FastAPI DELETE Method

## Concept

The **DELETE method is used to remove a resource from the server**.

Important characteristics:

* Removes existing data
* Usually uses **path parameters** to identify the resource
* Does not require a request body

Typical real-world use cases:

* Delete user account
* Remove product from inventory
* Cancel order

---

# 6. Example — DELETE API

```python
from fastapi import FastAPI

app = FastAPI()

items = {
    1: {"name": "Laptop", "price": 50000},
    2: {"name": "Phone", "price": 20000}
}

@app.delete("/items/{item_id}")
def delete_item(item_id: int):

    if item_id in items:
        deleted_item = items.pop(item_id)

        return {
            "message": "Item deleted",
            "deleted_item": deleted_item
        }

    return {"error": "Item not found"}
```

---

# 7. How the DELETE Request Works

### Step 1 — Client Sends Request

```
DELETE /items/2
```

---

### Step 2 — Server Removes Item

The server checks whether the item exists.

If found:

* It removes the item
* Returns confirmation

---

### Step 3 — Response Example

```json
{
 "message": "Item deleted",
 "deleted_item": {
  "name": "Phone",
  "price": 20000
 }
}
```

---

# 8. Complete Example — GET, POST, PUT, DELETE Together

```python
from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI()

items = {}

class Item(BaseModel):
    name: str
    price: float

# CREATE
@app.post("/items/{item_id}")
def create_item(item_id: int, item: Item):
    items[item_id] = item
    return {"message": "Item created", "data": item}

# READ
@app.get("/items")
def get_items():
    return items

# UPDATE
@app.put("/items/{item_id}")
def update_item(item_id: int, item: Item):
    items[item_id] = item
    return {"message": "Item updated", "data": item}

# DELETE
@app.delete("/items/{item_id}")
def delete_item(item_id: int):

    if item_id in items:
        removed = items.pop(item_id)
        return {"message": "Item deleted", "data": removed}

    return {"error": "Item not found"}
```

---

# 9. PUT vs DELETE (Key Differences)

| Feature        | PUT                         | DELETE              |
| -------------- | --------------------------- | ------------------- |
| Purpose        | Update existing resource    | Remove resource     |
| Request Body   | Usually required            | Not required        |
| Resource state | Resource continues to exist | Resource is removed |
| Idempotent     | Yes                         | Yes                 |

# 12. Real-Life Analogy

Consider an **online shopping system**.

### PUT Request

Customer updates their shipping address.

The old address is **replaced with the new address**.

This is similar to a **PUT request (update data)**.

---

### DELETE Request

Customer cancels an order.

The order is **removed from the system**.

This is similar to a **DELETE request (remove data)**.

---

# 13. Key Takeaways

* **PUT** updates an existing resource
* **DELETE** removes a resource
* PUT requests usually contain JSON request bodies
* DELETE requests usually use path parameters
* Both PUT and DELETE are **idempotent operations**
