# FastAPI Basics: Path Parameters and Query Parameters

---

# 1. Introduction to FastAPI

**FastAPI** is a modern, high‑performance Python web framework used to build APIs quickly and efficiently.

It is widely used in:

* Machine Learning APIs
* Data Science applications
* Backend microservices
* Production AI systems

FastAPI is built on top of:

* **Starlette** (for web handling)
* **Pydantic** (for data validation)

Key advantages:

| Feature              | Description                                  |
| -------------------- | -------------------------------------------- |
| High Performance     | Comparable to NodeJS and Go                  |
| Automatic Validation | Uses Python type hints                       |
| Auto API Docs        | Swagger UI and ReDoc generated automatically |
| Easy to Learn        | Simple syntax                                |

---

# 2. Minimal FastAPI Application

```python
from fastapi import FastAPI

app = FastAPI()

@app.get("/")
def read_root():
    return {"message": "Hello World"}
```

---

# 3. Code Explanation (Line by Line)

## 3.1 Import FastAPI

```python
from fastapi import FastAPI
```

This imports the **FastAPI class**, which is used to create the API application.

---

## 3.2 Create the Application Instance

```python
app = FastAPI()
```

This creates the **main API application object**.

All routes and endpoints will be attached to this object.

---

## 3.3 Create a Route (Path Operation)

```python
@app.get("/")
```

This is called a **Path Operation Decorator**.

Meaning:

When a **GET request** is sent to `/`, the function below will execute.

---

## 3.4 Define the Function

```python
def read_root():
```

This function handles the incoming request.

---

## 3.5 Return the Response

```python
return {"message": "Hello World"}
```

FastAPI automatically converts Python dictionaries into **JSON responses**.

Response:

```json
{
  "message": "Hello World"
}
```

# 6. Path Parameters

## What are Path Parameters?

**Path parameters are dynamic values that are part of the URL path.**

Example:

```
/users/10
```

Here:

```
10 → path parameter
```

---

# 7. Basic Path Parameter Example

```python
from fastapi import FastAPI

app = FastAPI()

@app.get("/users/{user_id}")
def get_user(user_id: int):
    return {"user_id": user_id}
```

---

# 8. How Path Parameters Work

If the user visits:

```
/users/5
```

FastAPI internally executes:

```python
get_user(user_id=5)
```

Response:

```json
{
  "user_id": 5
}
```

---

# 9. Automatic Type Validation

Example:

```python
def get_user(user_id: int):
```

If the user tries:

```
/users/abc
```

FastAPI automatically returns an error because `abc` is not an integer.

Example error:

```json
{
  "detail": [
    {
      "type": "int_parsing",
      "msg": "Input should be a valid integer"
    }
  ]
}
```

---

# 10. Multiple Path Parameters

```python
@app.get("/users/{user_id}/orders/{order_id}")
def get_order(user_id: int, order_id: int):
    return {
        "user_id": user_id,
        "order_id": order_id
    }
```

Example URL:

```
/users/5/orders/100
```

Response:

```json
{
  "user_id": 5,
  "order_id": 100
}
```

# 11. Query Parameters

## What are Query Parameters?

Query parameters are values passed **after `?` in the URL**.

Example:

```
/products?limit=10
```

Here:

```
limit → query parameter
```

---

# 12. Basic Query Parameter Example

```python
@app.get("/products")
def get_products(limit: int):
    return {"limit": limit}
```

Request:

```
/products?limit=10
```

Response:

```json
{
  "limit": 10
}
```

---

# 13. Multiple Query Parameters

```python
@app.get("/products")
def get_products(limit: int, skip: int):
    return {
        "limit": limit,
        "skip": skip
    }
```

Request:

```
/products?limit=10&skip=5
```

Response:

```json
{
  "limit": 10,
  "skip": 5
}
```

---

# 14. Optional Query Parameters

Query parameters can have **default values**, making them optional.

```python
@app.get("/products")
def get_products(limit: int = 10):
    return {"limit": limit}
```

Now the request can be:

```
/products
```

Response:

```json
{
  "limit": 10
}
```

---

# 15. Query Parameter with None

```python
@app.get("/items")
def get_items(q: str = None):
    return {"query": q}
```

Request:

```
/items?q=laptop
```

Response:

```json
{"query": "laptop"}
```

Request without query parameter:

```
/items
```

Response:

```json
{"query": null}
```

---

# 16. Path vs Query Parameters

| Feature  | Path Parameter               | Query Parameter      |
| -------- | ---------------------------- | -------------------- |
| Location | Inside URL path              | After `?`            |
| Example  | `/users/10`                  | `/users?limit=10`    |
| Purpose  | Identify a specific resource | Filtering, searching |
| Required | Usually required             | Usually optional     |

---

# 17. Example Using Both

```python
@app.get("/users/{user_id}")
def get_user(user_id: int, details: bool = False):
    return {
        "user_id": user_id,
        "details": details
    }
```

Example request:

```
/users/10?details=true
```

Response:

```json
{
  "user_id": 10,
  "details": true
}
```

# 18. Real Example (Typical API)

```python
from fastapi import FastAPI

app = FastAPI()

@app.get("/products/{product_id}")
def get_product(product_id: int, discount: bool = False):

    if discount:
        return {
            "product_id": product_id,
            "price": 90
        }

    return {
        "product_id": product_id,
        "price": 100
    }
```

Requests:

```
/products/5

/products/5?discount=true
```
