# FastAPI + MongoDB CRUD Application

----------------------------------------------------

# 1. Introduction

This project demonstrates how to build a **CRUD API using FastAPI and MongoDB**.

The application allows users to:

* Create users
* Read users
* Update users
* Delete users

This architecture is commonly used in:

* Backend services
* Machine learning APIs
* MLOps pipelines
* Microservices

---

# 2. What is CRUD

CRUD represents the **four basic database operations**.

| Operation | Meaning              | Example      |
| --------- | -------------------- | ------------ |
| Create    | Insert new data      | Add new user |
| Read      | Retrieve data        | Get users    |
| Update    | Modify existing data | Change email |
| Delete    | Remove data          | Delete user  |

Example record:

```json
{
  "name": "John",
  "email": "john@gmail.com",
  "age": 28
}
```

# 7. Pydantic Schema

FastAPI uses **Pydantic models for validation**.

## schemas.py

```python
from pydantic import BaseModel, EmailStr

class User(BaseModel):
    name: str
    email: EmailStr
    age: int
```

Example request body:

```json
{
  "name": "Rahul",
  "email": "rahul@gmail.com",
  "age": 25
}
```

If email format is incorrect, FastAPI automatically throws a validation error.

# 8. MongoDB Serialization

MongoDB returns `_id` which is not JSON serializable.

Therefore we convert it.

## models.py

```python
def user_serializer(user) -> dict:
    return {
        "id": str(user["_id"]),
        "name": user["name"],
        "email": user["email"],
        "age": user["age"]
    }


def users_serializer(users) -> list:
    return [user_serializer(user) for user in users]
```

Example MongoDB document:

```json
{
  "_id": "ObjectId",
  "name": "Rahul",
  "email": "rahul@gmail.com",
  "age": 25
}
```

Converted API response:

```json
{
  "id": "6534abc12",
  "name": "Rahul",
  "email": "rahul@gmail.com",
  "age": 25
}
```

# 9. FastAPI CRUD Implementation

## main.py

```python
from fastapi import FastAPI
from schemas import User
from database import user_collection
from models import user_serializer, users_serializer
from bson import ObjectId

app = FastAPI()
```

---

# CREATE Operation

Add a new user.

```python
@app.post("/users")
def create_user(user: User):

    new_user = dict(user)

    result = user_collection.insert_one(new_user)

    created_user = user_collection.find_one({"_id": result.inserted_id})

    return user_serializer(created_user)
```

Example request:

```
POST /users
```

Body:

```json
{
 "name": "Aman",
 "email": "aman@gmail.com",
 "age": 26
}
```

---

# READ Operation

## Get All Users

```python
@app.get("/users")
def get_users():

    users = user_collection.find()

    return users_serializer(users)
```

Example response:

```json
[
  {
    "id": "65432abc",
    "name": "Aman",
    "email": "aman@gmail.com",
    "age": 26
  }
]
```

---

## Get Single User

```python
@app.get("/users/{id}")
def get_user(id: str):

    user = user_collection.find_one({"_id": ObjectId(id)})

    return user_serializer(user)
```

Example request:

```
GET /users/65432abc
```

---

# UPDATE Operation

```python
@app.put("/users/{id}")
def update_user(id: str, user: User):

    user_collection.find_one_and_update(
        {"_id": ObjectId(id)},
        {"$set": dict(user)}
    )

    updated_user = user_collection.find_one({"_id": ObjectId(id)})

    return user_serializer(updated_user)
```

Example request:

```
PUT /users/65432abc
```

Body:

```json
{
 "name": "Aman Sharma",
 "email": "aman@gmail.com",
 "age": 27
}
```

MongoDB internally uses:

```
$set
```

Meaning:

Update specific fields.

---

# DELETE Operation

```python
@app.delete("/users/{id}")
def delete_user(id: str):

    user_collection.find_one_and_delete(
        {"_id": ObjectId(id)}
    )

    return {"message": "User deleted successfully"}
```

Example request:

```
DELETE /users/65432abc
```

Response:

```json
{
 "message": "User deleted successfully"
}
```

# 15. Limitations

| Limitation              | Explanation                      |
| ----------------------- | -------------------------------- |
