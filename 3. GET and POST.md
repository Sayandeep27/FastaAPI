# FastAPI GET and POST Methods

---

# 1. Introduction

When a client (browser, frontend app, mobile app, etc.) communicates with a **FastAPI server**, it sends an **HTTP request**.

Different **HTTP methods** tell the server what type of action the client wants to perform.

FastAPI supports all standard HTTP methods such as:

| Method | Purpose                                   |
| ------ | ----------------------------------------- |
| GET    | Retrieve data from the server             |
| POST   | Send data to the server / create new data |
| PUT    | Update entire data                        |
| PATCH  | Update partial data                       |
| DELETE | Delete data                               |

In this guide we will focus on the two most commonly used methods:

* **GET**
* **POST**

---

# 2. FastAPI GET Method

## Concept

The **GET method is used to retrieve data from the server**.

Important characteristics:

* It **does not modify data**
* It is used **only for reading data**
* Data is usually passed through:

  * **Path parameters**
  * **Query parameters**

Typical real‑world use cases:

* Retrieve a list of users
* Get product details
* Fetch weather information
* View blog posts

---

# 3. Example 1 — Simple GET API

```python
from fastapi import FastAPI

app = FastAPI()

@app.get("/")
def home():
    return {"message": "Welcome to FastAPI"}
```

## Explanation

The decorator:

```
@app.get("/")
```

means:

When someone sends a **GET request to the root URL**, FastAPI will execute the `home()` function.

Example request:

```
http://127.0.0.1:8000/
```

Example response:

```json
{
 "message": "Welcome to FastAPI"
}
```

---

# 4. Example 2 — GET with Path Parameters

Path parameters are variables that appear **inside the URL path**.

Example URL:

```
/items/10
```

## Code Example

```python
from fastapi import FastAPI

app = FastAPI()

@app.get("/items/{item_id}")
def get_item(item_id: int):
    return {"item_id": item_id}
```

## How It Works

* `{item_id}` is a **path parameter**
* FastAPI automatically converts it to the correct data type

Example request:

```
http://127.0.0.1:8000/items/5
```

Example response:

```json
{
 "item_id": 5
}
```

---

# 5. Example 3 — GET with Query Parameters

Query parameters appear **after the `?` symbol in a URL**.

Example URL:

```
/products?category=laptop&price=50000
```

## Code Example

```python
from fastapi import FastAPI

app = FastAPI()

@app.get("/products")
def get_products(category: str, price: int):
    return {
        "category": category,
        "price": price
    }
```

## Request Example

```
http://127.0.0.1:8000/products?category=mobile&price=20000
```

## Response

```json
{
 "category": "mobile",
 "price": 20000
}
```

---

# 6. FastAPI POST Method

## Concept

The **POST method is used to send data to the server to create new resources**.

Common use cases:

* Create a user
* Add a product
* Register a student
* Upload information

Unlike GET requests, POST requests send data inside the **request body**, usually in **JSON format**.

---

# 7. Example — POST API

```python
from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI()

class User(BaseModel):
    name: str
    age: int

@app.post("/users")
def create_user(user: User):
    return {
        "message": "User created",
        "data": user
    }
```

---

# 8. Request Body (JSON)

When calling the API, we send JSON data in the request body:

```json
{
 "name": "John",
 "age": 25
}
```

---

# 9. Response Example

```json
{
 "message": "User created",
 "data": {
   "name": "John",
   "age": 25
 }
}
```

---

# 10. Detailed Explanation

## Step 1 — Create a Pydantic Model

```python
class User(BaseModel):
    name: str
    age: int
```

This model defines the **structure of the incoming request data**.

FastAPI automatically validates the request against this schema.

---

## Step 2 — Define the POST Endpoint

```python
@app.post("/users")
```

This means:

A **POST request to `/users`** will trigger this function.

---

## Step 3 — Receive Request Data

```python
def create_user(user: User):
```

FastAPI will automatically:

1. Read the JSON body
2. Convert it into a **User object**
3. Validate the data

---

# 11. Complete Example — GET and POST Together

```python
from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI()

items = []

class Item(BaseModel):
    name: str
    price: float

# GET method
@app.get("/items")
def get_items():
    return items

# POST method
@app.post("/items")
def create_item(item: Item):
    items.append(item)
    return {
        "message": "Item added",
        "item": item
    }
```

---

# 12. How the API Works

## Step 1 — Add Item (POST)

Request:

```
POST /items
```

Request body:

```json
{
 "name": "Laptop",
 "price": 50000
}
```

---

## Step 2 — Retrieve Items (GET)

Request:

```
GET /items
```

Response:

```json
[
 {
  "name": "Laptop",
  "price": 50000
 }
]
```

---

# 13. GET vs POST (Important Differences)

| Feature                     | GET             | POST                 |
| --------------------------- | --------------- | -------------------- |
| Purpose                     | Retrieve data   | Create or send data  |
| Data Location               | URL parameters  | Request body         |
| Security for sensitive data | Not recommended | Safer than GET       |
| Caching                     | Supported       | Not typically cached |
| Idempotent                  | Yes             | No                   |

# 16. Real‑Life Analogy

Consider a **restaurant system**.

### GET Request

Customer asks:

```
Show me the menu
```

The server returns the menu.

This is similar to a **GET request (retrieve data)**.

---

### POST Request

Customer places an order:

```
Order pizza
```

The restaurant creates a **new order**.

This is similar to a **POST request (create data)**.

# 17. Key Takeaways

* **GET** is used to retrieve information from the server
* **POST** is used to send data and create new resources
* FastAPI automatically validates data using **Pydantic models**
* Path parameters and query parameters are commonly used with GET
* JSON request bodies are commonly used with POST
