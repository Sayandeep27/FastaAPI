# 1. Introduction

FastAPI is a **modern Python web framework** used for building **high-performance APIs**.

However, FastAPI alone cannot store data permanently.

To store and retrieve data, we connect FastAPI with a **database**.

Two commonly used databases are:

| Database                     | Type                    |
| ---------------------------- | ----------------------- |
| MSSQL (Microsoft SQL Server) | Relational Database     |
| MongoDB                      | NoSQL Document Database |

This guide explains **how to connect both databases with FastAPI in simple steps**.

---

# 2. Why Connect a Database with FastAPI

FastAPI handles:

* API endpoints
* Request handling
* Response generation
* Data validation
* Business logic

But data must be **stored somewhere permanently**.

Example workflow:

```
Client (React / Postman)
        |
        v
     FastAPI
        |
        v
     Database
```

Example scenario:

User registers → FastAPI receives request → data saved in database.

---

# 3. FastAPI + MSSQL

MSSQL is a **Relational Database Management System (RDBMS)**.

Data is stored in **tables with rows and columns**.

Example table:

| id | name  | age |
| -- | ----- | --- |
| 1  | John  | 25  |
| 2  | Alice | 30  |

To connect FastAPI with MSSQL we typically use:

| Tool       | Purpose                            |
| ---------- | ---------------------------------- |
| SQLAlchemy | ORM to interact with SQL databases |
| pyodbc     | ODBC driver for MSSQL              |

# 5. Creating MSSQL Database Connection

Create a file:

```
database.py
```

```python
from sqlalchemy import create_engine
from sqlalchemy.orm import sessionmaker

DATABASE_URL = "mssql+pyodbc://username:password@localhost:1433/mydatabase?driver=ODBC+Driver+17+for+SQL+Server"

engine = create_engine(DATABASE_URL)

SessionLocal = sessionmaker(
    autocommit=False,
    autoflush=False,
    bind=engine
)
```

Connection Flow:

```
FastAPI -> SQLAlchemy -> pyodbc -> MSSQL
```

SQLAlchemy generates SQL queries and sends them to MSSQL.

---

# 6. Creating Table Model

Create:

```
models.py
```

```python
from sqlalchemy import Column, Integer, String
from sqlalchemy.orm import declarative_base

Base = declarative_base()

class User(Base):

    __tablename__ = "users"

    id = Column(Integer, primary_key=True, index=True)
    name = Column(String)
    age = Column(Integer)
```

This class represents the **database table structure**.

---

# 7. Creating FastAPI Application

Create:

```
main.py
```

```python
from fastapi import FastAPI, Depends
from sqlalchemy.orm import Session

from database import SessionLocal
import models

app = FastAPI()
```

---

# 8. Database Session Dependency

FastAPI creates a **database session per request**.

```python
def get_db():

    db = SessionLocal()

    try:
        yield db
    finally:
        db.close()
```

Purpose:

| Step            | Explanation         |
| --------------- | ------------------- |
| Session created | New DB connection   |
| Request handled | Query executed      |
| Session closed  | Connection released |

---

# 9. Create Data (INSERT)

```python
@app.post("/users")
def create_user(name: str, age: int, db: Session = Depends(get_db)):

    user = models.User(name=name, age=age)

    db.add(user)
    db.commit()
    db.refresh(user)

    return user
```

Explanation:

```
1 request received
2 user object created
3 added to database session
4 commit saves data permanently
```

---

# 10. Read Data (SELECT)

```python
@app.get("/users")
def get_users(db: Session = Depends(get_db)):

    users = db.query(models.User).all()

    return users
```

This returns all users stored in the table.

# 12. FastAPI + MongoDB

MongoDB is a **NoSQL document database**.

Instead of tables, it stores **JSON-like documents**.

Example document:

```
{
  "name": "John",
  "age": 25
}
```

MongoDB structure:

```
Database
   |
   |-- Collection
           |
           |-- Documents
```

# 16. Create Data in MongoDB

```python
@app.post("/users")
def create_user(user: dict):

    result = collection.insert_one(user)

    return {"id": str(result.inserted_id)}
```

Example Request:

```
POST /users

{
  "name": "John",
  "age": 25
}
```

---

# 17. Read Data from MongoDB

```python
@app.get("/users")
def get_users():

    users = []

    for user in collection.find():

        user["_id"] = str(user["_id"])
        users.append(user)

    return users
```

Why convert `_id` to string?

MongoDB `_id` is **ObjectId**, which cannot be returned directly as JSON.

# 21. MSSQL vs MongoDB Comparison

| Feature         | MSSQL           | MongoDB        |
| --------------- | --------------- | -------------- |
| Database Type   | Relational      | NoSQL          |
| Data Format     | Tables          | JSON Documents |
| Schema          | Fixed           | Flexible       |
| Query Language  | SQL             | Mongo Query    |
| FastAPI Library | SQLAlchemy      | PyMongo        |
| Best For        | Structured Data | Flexible Data  |

---

# 22. When to Use MSSQL

Use MSSQL when your application requires:

* Financial transactions
* Banking systems
* Strong data consistency
* Complex SQL joins
* Structured data

Examples:

* Banking applications
* ERP systems
* Inventory systems

---

# 23. When to Use MongoDB

Use MongoDB when you need:

* Flexible schema
* Rapid development
* JSON data storage
* High scalability

Examples:

* Logging systems
* Analytics systems
* Content management
* Real-time applications

# 24. Production Level Setup (Industry Practice)

In real-world projects developers use improved setups.

### MSSQL Production Stack

```
FastAPI
SQLAlchemy
Alembic (database migrations)
Pydantic schemas
Dependency Injection
```

### MongoDB Production Stack

```
FastAPI
Motor (async MongoDB driver)
Pydantic models
Dependency Injection
```

Motor is preferred because FastAPI supports **asynchronous programming**.
